# Веб-додаток для офісного перекладу — ТЗ і План

## Архітектура (остаточне рішення)

**Гібрид з трьох джерел:**

| Звідки | Що беремо |
|--|--|
| **translate_server.py** | FastAPI бекенд, черга з тікетами, зупинка, конфіг, українська локалізація, 34 мови |
| **Translate_word_to_Word.py** (desktop) | Логіка збереження формату DOCX/PDF, чанкування, маркери `⟨P⟩`/`⟨N⟩` |
| **local-translator** (стиль) | Мінімалістичний сучасний UI, drag&drop |

---

## Інфраструктура

- Сервер: Ubuntu 26.04, i5-12500, RTX 3060 12GB, 32GB RAM
- IP: `192.168.88.59`
- Ollama: `192.168.88.59:11434`
- Веб-додаток: порт 7860 (FastAPI)
- Модель: `rinex20/translategemma3` (6.9GB, Gemma3 12B)
- До 5 користувачів одночасно
- Розгортання: `.py` файли + systemd service (git pull → restart)
- Платформа клієнта: тільки ПК (без мобільної адаптації)

## Користувацький інтерфейс

Мінімалістична одна сторінка у стилі local-translator:

1. **Вибір мови** з / на (34 мови + автовизначення)
2. **Поле тексту** + кнопка "Перекласти" + "Зупинити"
3. **Drag&drop файлу** (DOCX/PDF/TXT) + кнопка "Перекласти файл" + "Зупинити"
4. **Результат:**
   - Для тексту — поле з результатом
   - Для файлу — кнопка "Скачати" (у тому ж форматі що завантажений)

**Прибрано:** вибір моделі, авторизація, стрімінг, thinking, лог, налаштування, прогрес-бар

**Залишено:** мови, текст/файл, результат, зупинка

---

## Адмін-режим (`/admin`)

Захищено **Basic Auth** (логін/пароль через стандартне вікно браузера).

Що є на сторінці:
- **Активні запити** — хто, що, скільки часу йде
- **Лог з timestamp** — всі дії і помилки (повний traceback)
- **Стрімінг токенів + thinking** — режим відладки моделі
- **Налаштування:**
  - URL Ollama
  - Модель
  - Макс. токенів
  - Таймаут
  - Розмір чанка
  - Системне повідомлення
  - Шаблон промпту
  - **Ліміти файлів** (макс. сторінок, макс. символів)
  - Логін і пароль для адміна
- **Статистика** (див. нижче)

Налаштування зберігаються у `translator_config.json`.

---

## Обробка помилок

**Користувачу** — дружнє повідомлення без технічних деталей:
- "Файл пошкоджено або непідтримуваний формат"
- "Помилка сервера. Спробуйте пізніше"
- "Файл занадто великий" (при перевищенні ліміту)
- "Переклад зупинено"

**В лог (адмін):**
- Повний traceback
- Timestamp
- IP / умовний користувач
- Параметри запиту (мови, розмір файлу)

---

## Статистика

Зберігається в **SQLite** (один файл `stats.db`, без додаткових сервісів).

**Що збираємо:**
- IP клієнта
- Тип запиту (текст / файл)
- Назва файлу (якщо файл)
- Мови (з → на)
- Кількість символів
- Кількість сторінок (для PDF/DOCX)
- Час перекладу (секунд)
- Статус (успіх / помилка / зупинка)
- Timestamp

**Відображення на `/admin`:**
- Таблиця останніх N запитів
- Загальні цифри: всього перекладів, символів, годин роботи
- Топ користувачів за активністю

## Ліміти

За замовчуванням:
- **10 сторінок** для PDF/DOCX
- **30 000 символів** для тексту і TXT файлів

Налаштовуються в `/admin`. Якщо перевищено — користувач отримує повідомлення "Файл занадто великий (максимум N сторінок)".

---

## Збереження формату файлу

- **DOCX → DOCX** — логіка з `Translate_word_to_Word.py:run_translation` (python-docx)
- **PDF → PDF** — логіка з `Translate_word_to_Word.py:run_translation_pdf` (PyMuPDF: redact + insert_textbox)
- **TXT → TXT** — вже працює

## План реалізації

### Етап 1 — Перенесення на сервер
1. Завантажити модель: `ollama pull rinex20/translategemma3`
2. Скопіювати `translate_server.py` на сервер через git
3. Оновити `DEFAULTS`: URL → `http://192.168.88.59:11434/v1`, модель → `rinex20/translategemma3`
4. Створити systemd service
5. Доступ через `http://192.168.88.59:7860`
6. Перевірити що поточна версія працює

### Етап 2 — Розділення на користувацький і адмін UI
1. Створити новий мінімалістичний HTML для користувачів (стиль local-translator)
2. Перенести існуючий UI на `/admin` з Basic Auth
3. Налаштувати маршрутизацію FastAPI

### Етап 3 — Збереження формату файлів
1. Перенести логіку DOCX перекладу з desktop app
2. Перенести логіку PDF перекладу з PyMuPDF
3. Замінити поточний `/translate-file` на диспетчер за розширенням
4. Результат скачується у тому ж форматі

### Етап 4 — Помилки, ліміти, статистика
1. Створити обробник помилок з дружніми повідомленнями
2. Лог помилок з повним traceback
3. Перевірка лімітів на завантаженні (сторінки, символи)
4. SQLite база `stats.db`
5. Таблиця статистики на `/admin`

### Етап 5 — Тестування
1. Тест з 3-5 одночасними користувачами
2. Перевірка великих файлів і лімітів
3. Перевірка помилок (пошкоджені файли, недоступна Ollama)
4. Виправлення проблем

## Чому саме цей підхід

**translate_server.py** дає готовий бекенд:
- Не треба переписувати чергу
- Не треба переписувати конфіг
- Українська локалізація є

**Desktop app** дає перевірену логіку формату:
- DOCX → DOCX з форматуванням працює
- PDF → PDF з layout працює (PyMuPDF redact+reinsert)
- Чанкування з маркерами працює

**local-translator** дає стиль UI:
- Сучасний вигляд
- Drag&drop
- Мінімалізм

**Не повторюємо помилок local-translator:**
- НЕ переносимо парсинг файлів у браузер
- НЕ викидаємо чергу і зупинку
- НЕ викидаємо адмін-можливості

## Відхилені варіанти

- ❌ **local-translator як основа** — додавання адмін-режиму вимагає бекенд

- systemd service на Ubuntu
